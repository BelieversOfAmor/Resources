Mina学习笔记(5) MINA中的Session

session就像mina框架的心脏，没一个client的连接到达server之后都会创建一个新的session，一直保存在内存单元中直到client 连接断开。

session 常常用来储存一些和连接相关的持久化信息，这些信息在server 进行处理的时候都是非常有用的，生命周期为整个会话

1、session的状态

通常来讲 mina 的session 有以下几个状态

1、Connected : client 连接上server 之后，这时 session 已经建立，并且该session 是有效的、可用的

2、Idle   : 此时 session并没有处理任何的请求，并且这个时间超过了你设定的 session 进入空闲状态的那个时间点，idle又细分为三个状态

  (1) idle for read  超过读通道设定的空闲时间

  (2) idle for write 超过写通道设定的空闲时间

  (3) idle for both  读写通道均超过设定空闲时间

3、Closing : 这时候session即将被关闭但是有没有完全关闭(对余下未处理完的消息进行 flushed 操作)

4、Closed : session已经完全关闭，并且这个状态时不可逆的

IoAcceptor acceptor = new NioSocketAcceptor();
				acceptor.getSessionConfig().setIdleTime(IdleStatus.WRITER_IDLE,10); //写 通道均在10 秒内无任何操作就进入空闲状态
				acceptor.getSessionConfig().setIdleTime(IdleStatus.READER_IDLE,10); //读 通道均在10 秒内无任何操作就进入空闲状态
				acceptor.getSessionConfig().setIdleTime(IdleStatus.BOTH_IDLE,10);   //读写 通道均在10 秒内无任何操作就进入空闲状态

2、session常用配置

一般地在创建一个连接时需要对一个session做一些初始化

   // 创建一个非阻塞的server端的Socket
				IoAcceptor acceptor = new NioSocketAcceptor();
				acceptor.getSessionConfig().setIdleTime(IdleStatus.BOTH_IDLE,10);   //读写 通道均在10 秒内无任何操作就进入空闲状态
				acceptor.getSessionConfig().setReadBufferSize(1024); //  设置读写缓冲区大小
				acceptor.getSessionConfig().setWriteTimeout(1000); // 这个方法设置写操作的超时时间

3、管理 session公共属性

有时在server端的session需要维护的一些以后我们将要用到的信息，这些信息可能经常用到，那么这时候mina提供 key-value 这种数据形式来对这类数据进行保存，这样的话我们就能保存任何数据类型的信息了，这样很方便我们进行开发,那么在下次需要用到的时候直接用key就能取到和key对应的值了，这个web开发的时候的session是同一种概念

int counterValue = session.getAttribute( "counter" );
session.setAttribute( "counter", counterValue + 1 );

这样我们就能对很方便的读写操作session中相应的属性值了

当然存放key-value 的seesion容器在session创建的时候就自动初始化了，在session停止的时候又自动销毁了

4、自定义session容器

正如上面所说的那样，session 其实就是一个key-value的容器，默认是map实现的，但是这时候当用户需要长期的持久化一些信息或者这些信息放在内存中又太大了，那么这时候这种key-value的数据结构就不能满足需求了，幸好mina给了我们解决办法，可自定义session容器(在session建立的时候就需要初始化)

自动以sessiion需要注意两点:1.自定义容易必须在session初始化的时候就建立 2.实现 IoSessionDataStructureFactory 接口自定义session容器操作

初始化session容器

protected final void initSession(IoSession session,
        IoFuture future, IoSessionInitializer sessionInitializer) {
    ...
    try {
        ((AbstractIoSession) session).setAttributeMap(session.getService()
                .getSessionDataStructureFactory().getAttributeMap(session));
    } catch (IoSessionInitializationException e) {
        throw e;
    } catch (Exception e) {
        throw new IoSessionInitializationException(
                "Failed to initialize an attributeMap.", e);
    }

自定义session容器

public interface IoSessionDataStructureFactory {
    /**
     * Returns an {@link IoSessionAttributeMap} which is going to be associated
     * with the specified <tt>session</tt>.  Please note that the returned
     * implementation must be thread-safe.
     */
     IoSessionAttributeMap getAttributeMap(IoSession session) throws Exception;
 }

5、session过滤

严格来讲每隔session都有一个过滤器链，这些过滤器可以对请求传进来的参数和系统发送出去的参数进行过滤，这些过滤器都是针对单个session的，但是大多数时候我们都会用一个过滤器链对对存在的seesion进行过滤，不管怎样对可以对特殊的seesion定义过滤器，比如，我们可以只对某一个特定的session加入一个特定的log

6、session统计功能

 每个session在运行期间都会有一些的记录，那么可以通过一些特定方法进行获取

1、接收和发送消息的bytes

2、接收和发送消息的数量

3、读写通道的状态

4、吞吐量

7、业务处理

最后哦，也是最后总要的一点，一个session 有很重要的一个业务操作，session还负责在你的应用程序中发送消息，发送数据包，当然这一系列的事我们只需简单的调用write() 方法就可以了

session.write(Object message); // 向远程连接发送一条数据
